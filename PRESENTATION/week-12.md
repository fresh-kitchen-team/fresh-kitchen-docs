# 12주차 발표

Status: draft
Updated: 2026-05-22

## Purpose

12주차 발표는 저번 중간발표에서 보여주지 못한 화면과 기능을 실제 흐름으로 소개하는 시연입니다.

개발 스코프는 12주차 발표 전까지 개발하거나 연결해야 하는 범위이고, 시연 시나리오는 발표에서 보여줄 화면과 기능의 순서입니다.

## Presentation Goal

이번 발표의 주장은 아래처럼 가져갑니다.

- FreshKitchen은 로그인, 계정 설정, 문의 메일, AI 채팅, 유통기한 관리까지 이어지는 사용자 흐름을 갖춘 서비스입니다.
- 12주차 발표는 전체 기능 완성 여부를 검증하는 자리가 아니라, 이전에 보여주지 못했던 핵심 화면과 기능을 소개하는 자리입니다.
- 개발 스코프는 파트별로 구분하고, 시연은 사용자가 따라가는 화면 흐름으로 보여줍니다.

## Development Scope

12주차 발표 전까지 개발하거나 연결해야 하는 범위입니다.

### 프론트

- 구글 로그인 진입 화면을 준비합니다.
- 설정 화면에서 계정 확인과 로그아웃 흐름을 준비합니다.
- AI 채팅 설정 화면에서 문의하기 흐름을 준비합니다.
- 문의하기는 앱 안에서 이력을 보여주지 않고, 실제 메일 수신 여부를 확인하는 방식으로 시연합니다.
- AI 채팅 화면에서 추가 정보 제공 질문과 추천 기준 기반 레시피 생성 흐름을 준비합니다.
- 유통기한 설정 화면에서 `D-1`, `D-3`, `D-5`, `D-10` 버튼을 준비합니다.

### 백엔드

- 구글 로그인은 구현 완료 상태이며, 프론트 로그인 진입과 연결해 설명할 수 있어야 합니다.
- 로그아웃을 구현합니다.
- Java Mail Sender로 유저 계정에 문의 메일을 보내는 흐름을 설명할 수 있어야 합니다.
- 분석/팁 화면에 연결할 백엔드 응답 또는 라우팅 흐름을 준비합니다.

### AI

- AI 채팅에서 추가 정보 제공 질문 흐름을 준비합니다.
- 추천 기준과 알림 설정을 반영한 레시피 생성 흐름을 준비합니다.
- 냉장고 detection 결과를 백엔드와 연결하는 작업을 시작합니다.

### 공통/연결

- 백엔드 구현 완료 기능과 프론트 연결 상태는 구분해서 설명합니다.
- 프론트 연결이 완전하지 않은 구간은 화면, 요청 흐름, 준비된 예시로 설명합니다.
- 실시간 응답이 불안정한 기능은 준비된 질문과 응답 예시로 대체할 수 있게 합니다.

## Demo Scope

이번 발표에서 보여줄 화면과 기능입니다.

- 로그인 화면의 구글 로그인 진입
- 설정 화면의 계정 확인과 로그아웃
- 로그아웃 후 재로그인과 홈 화면 진입
- AI 채팅 설정 화면의 문의하기와 실제 메일 수신 확인
- AI 채팅 설정 화면의 `추가 정보 제공` 질문
- 추천 기준과 `레시피 생성 완료 알림` 설정 후 레시피 답변 확인
- 식재료 scan 화면의 `D-1`, `D-3`, `D-5`, `D-10` 빠른 입력 버튼

## Storyboard

1. 중간발표에서 부족했던 점을 짧게 짚습니다.
   - 지난 발표에서는 실제 사용자 흐름 시연이 충분하지 않았습니다.
   - 이번 발표에서는 이전에 보여주지 못한 화면과 기능을 순서대로 보여줍니다.
2. 로그인과 계정 설정 흐름을 소개합니다.
   - 로그인 화면에서 구글 로그인을 보여줍니다.
   - 설정 화면에서 계정을 확인하고 로그아웃합니다.
   - 다시 구글 로그인 후 홈 화면에 진입합니다.
3. AI 채팅 설정 화면의 문의하기 흐름을 소개합니다.
   - 문의하기를 통해 유저 계정에 메일을 보냅니다.
   - 앱 안의 문의 이력 화면은 시연하지 않고, 실제 메일 수신만 확인합니다.
4. AI 채팅 설정 화면의 추가 정보 제공 질문을 소개합니다.
   - `추가 정보 제공` 버튼을 누릅니다.
   - "김치찌개 1인분의 영양표가 궁금해" 질문을 보내고 답변을 확인합니다.
5. 추천 기준과 알림 설정 기반 레시피 답변을 소개합니다.
   - 추천 기준에서 `영양균형기반 추천`, `자주 사용하는 재료 우선` 버튼을 활성화합니다.
   - `레시피 생성 완료 알림`을 설정하고 알림 및 레시피 답변을 확인합니다.
6. 식재료 scan 화면의 유통기한 빠른 입력을 소개합니다.
   - 캘린더 입력이 아니라 `D-1`, `D-3`, `D-5`, `D-10` 버튼으로 입력할 수 있음을 보여줍니다.

## Demo Scenario

1. 로그인 화면에 진입합니다.
2. 구글 로그인 버튼을 눌러 로그인합니다.
3. 설정 화면으로 이동해 로그인된 계정을 확인합니다.
4. 설정 화면에서 로그아웃합니다.
5. 다시 구글 로그인 후 홈 화면 진입을 확인합니다.
6. AI 채팅 설정 화면으로 이동합니다.
7. 문의하기를 통해 유저 계정에 메일을 보냅니다.
8. 앱 안에서는 문의 이력을 보여주지 않고, 실제 메일 수신만 확인합니다.
9. AI 채팅 설정 화면에서 `추가 정보 제공` 버튼을 누릅니다.
10. "김치찌개 1인분의 영양표가 궁금해" 질문을 보내고 답변을 확인합니다.
11. 추천 기준에서 `영양균형기반 추천`, `자주 사용하는 재료 우선` 버튼을 활성화합니다.
12. `레시피 생성 완료 알림`을 설정합니다.
13. 알림과 레시피 답변을 확인합니다.
14. 식재료 scan 화면으로 이동합니다.
15. 캘린더 대신 `D-1`, `D-3`, `D-5`, `D-10` 버튼 4개로 유통기한을 입력할 수 있음을 시연합니다.

## Demo Explanation

- 구글 로그인은 실제 사용자가 서비스에 진입하는 첫 화면입니다.
- 설정 화면은 로그인된 계정 확인과 로그아웃 흐름을 보여주는 화면입니다.
- 문의하기는 앱 안에서 이력을 보여주지 않고, 유저 계정으로 실제 메일이 발송되는지 확인합니다.
- AI 채팅은 추가 정보 제공 질문과 추천 기준 기반 레시피 생성을 모두 보여주는 화면입니다.
- `영양균형기반 추천`, `자주 사용하는 재료 우선`은 레시피 답변의 추천 기준을 사용자가 선택하는 기능입니다.
- `레시피 생성 완료 알림`은 레시피 답변 완료를 사용자가 놓치지 않게 알려주는 설정입니다.
- `D-1`, `D-3`, `D-5`, `D-10` 버튼은 API 필드가 아니라 프론트 입력 편의 기능입니다.

## Fallback

로그인 연결이 불안정하면 로그인 화면과 백엔드 구글 로그인 구현 완료 상태를 설명합니다.

메일 발송이 실패하면 Java Mail Sender가 문의 메일을 보내는 흐름과 실제 메일 확인 기준을 설명합니다.

AI 채팅 실시간 응답이 실패하면 준비된 질문과 응답 예시로 대체합니다.

알림 시연이 실패하면 `레시피 생성 완료 알림` 설정 화면과 준비된 알림 결과 화면으로 대체합니다.

유통기한 버튼 저장까지 준비되지 않았으면 버튼 UI와 날짜 계산 의도만 보여줍니다.
