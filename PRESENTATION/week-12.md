# 12주차 발표

Status: draft
Updated: 2026-05-17

## Purpose

12주차 발표는 저번 중간발표에서 보여주지 못한 화면과 기능을 실제 흐름으로 소개하는 시연입니다.

개발 스코프는 12주차 발표 전까지 개발하거나 연결해야 하는 범위이고, 시연 시나리오는 발표에서 보여줄 화면과 기능의 순서입니다.

## Presentation Goal

이번 발표의 주장은 아래처럼 가져갑니다.

- FreshKitchen은 온보딩, 로그인, 시스템 알림, AI 채팅, 유통기한 관리까지 이어지는 사용자 흐름을 갖춘 서비스입니다.
- 12주차 발표는 전체 기능 완성 여부를 검증하는 자리가 아니라, 이전에 보여주지 못했던 핵심 화면과 기능을 소개하는 자리입니다.
- 개발 스코프는 파트별로 구분하고, 시연은 사용자가 따라가는 화면 흐름으로 보여줍니다.

## Development Scope

12주차 발표 전까지 개발하거나 연결해야 하는 범위입니다.

### 프론트

- 카카오/구글 로그인 진입 화면을 준비합니다.
- 온보딩 안에서 동의약관 렌더링 화면을 준비합니다.
- 시스템이 유저에게 이상 상황 메일을 보냈을 때 필요한 알림 상태 화면을 정리합니다.
  - 유저가 앱 안에서 알림 발송 상태를 확인할 화면이 필요한지 검토합니다.
  - 메일 수신 후 사용자가 돌아올 안내 화면이나 알림 히스토리 화면이 필요한지 검토합니다.
- AI 채팅 화면에서 레시피 추천과 일반 검색 질문 흐름을 준비합니다.
- 유통기한 설정 화면에서 `D-1`, `D-3`, `D-5`, `D-10` 버튼을 준비합니다.

### 백엔드

- 카카오/구글 로그인은 구현 완료 상태이며, 프론트 로그인 진입과 연결해 설명할 수 있어야 합니다.
- 로그아웃을 구현합니다.
- Java Mail Sender로 시스템이 유저에게 이상 상황 메일을 보내는 흐름을 설명할 수 있어야 합니다.
- 분석/팁 화면에 연결할 백엔드 응답 또는 라우팅 흐름을 준비합니다.

### AI

- AI 채팅에서 냉장고 내 식재료 기반 레시피 추천 흐름을 준비합니다.
- 일반 LLM 웹 검색 연동 질의 흐름을 준비합니다.
- 냉장고 detection 결과를 백엔드와 연결하는 작업을 시작합니다.

### 공통/연결

- 백엔드 구현 완료 기능과 프론트 연결 상태는 구분해서 설명합니다.
- 프론트 연결이 완전하지 않은 구간은 화면, 요청 흐름, 준비된 예시로 설명합니다.
- 실시간 응답이 불안정한 기능은 준비된 질문과 응답 예시로 대체할 수 있게 합니다.

## Demo Scope

이번 발표에서 보여줄 화면과 기능입니다.

- 온보딩 화면
- 카카오/구글 로그인 진입 화면
- 온보딩 동의약관 렌더링 화면
- 시스템 이상 상황 메일 발송 설명과 필요한 알림 화면 방향
- AI 채팅의 냉장고 내 식재료 기반 레시피 추천
- AI 채팅의 일반 LLM 웹 검색 연동
- 유통기한 설정 화면의 `D-1`, `D-3`, `D-5`, `D-10` 버튼

## Storyboard

1. 중간발표에서 부족했던 점을 짧게 짚습니다.
   - 지난 발표에서는 실제 사용자 흐름 시연이 충분하지 않았습니다.
   - 이번 발표에서는 이전에 보여주지 못한 화면과 기능을 순서대로 보여줍니다.
2. 온보딩과 로그인 화면을 소개합니다.
   - 사용자가 처음 진입하는 화면을 보여줍니다.
   - 카카오/구글 로그인 진입을 보여주고, 백엔드 로그인 구현 상태는 개발 스코프에서 분리해 설명합니다.
3. 동의약관 화면을 소개합니다.
   - 온보딩 과정에서 약관이 렌더링되는 화면을 보여줍니다.
4. 시스템 알림 메일 흐름을 소개합니다.
   - Java Mail Sender는 시스템이 유저에게 이상 상황을 알리는 메일 발송 기능이라고 설명합니다.
   - 이 흐름에 맞춰 앱 안에서 어떤 알림 상태 화면이 필요한지 설명합니다.
5. AI 채팅 화면을 소개합니다.
   - 냉장고 내 식재료 기반 레시피 추천 질문을 보여줍니다.
   - 일반 LLM 웹 검색 연동 질문을 보여줍니다.
6. 유통기한 빠른 입력 화면을 소개합니다.
   - `D-1`, `D-3`, `D-5`, `D-10` 버튼으로 만료일 입력 부담을 줄이는 화면을 보여줍니다.

## Demo Scenario

1. 온보딩 화면에 진입합니다.
2. 카카오/구글 로그인 버튼 또는 로그인 진입 화면을 보여줍니다.
3. 동의약관 렌더링 화면을 보여줍니다.
4. 시스템 이상 상황 메일 발송 흐름을 설명합니다.
5. 메일 수신 후 유저가 앱에서 확인해야 할 알림 상태 화면 방향을 설명합니다.
6. AI 채팅 화면으로 이동합니다.
7. 냉장고 내 식재료 기반 레시피 추천 질문을 보냅니다.
8. 일반 LLM 웹 검색 연동 질문을 보냅니다.
9. 식재료 등록 또는 유통기한 설정 화면으로 이동합니다.
10. `D-1`, `D-3`, `D-5`, `D-10` 버튼 4개를 사용해 유통기한 입력 화면을 보여줍니다.

## Demo Explanation

- 온보딩은 사용자가 서비스에 처음 진입하는 화면입니다.
- 동의약관은 온보딩 안에서 보여주는 사용자 진입 조건입니다.
- Java Mail Sender는 사용자가 문의를 보내는 기능이 아니라, 시스템이 유저에게 이상 상황을 알리는 메일 발송 흐름입니다.
- 프론트는 이 메일 흐름에 맞춰 알림 발송 상태, 알림 히스토리, 또는 메일 수신 후 복귀 화면이 필요한지 검토해야 합니다.
- AI 채팅은 냉장고 내 식재료를 활용한 추천과 일반 검색성 질문을 모두 보여주는 화면입니다.
- `D-1`, `D-3`, `D-5`, `D-10` 버튼은 API 필드가 아니라 프론트 입력 편의 기능입니다.

## Fallback

로그인 연결이 불안정하면 온보딩 화면과 백엔드 카카오/구글 로그인 구현 완료 상태를 설명합니다.

메일 발송이 실패하면 Java Mail Sender가 시스템 알림 메일을 보내는 흐름과 필요한 프론트 알림 화면 방향을 설명합니다.

AI 채팅 실시간 응답이 실패하면 준비된 질문과 응답 예시로 대체합니다.

유통기한 버튼 저장까지 준비되지 않았으면 버튼 UI와 날짜 계산 의도만 보여줍니다.
