# 16주차 최종 발표

Status: draft
Updated: 2026-05-17

## Purpose

16주차 최종 발표는 FreshKitchen의 문제 정의, 핵심 기능, 파트 간 연결 결과를 최종 흐름으로 보여줍니다.

문서, API 계약, 시연 흐름이 실제 발표 내용과 다르지 않게 맞추는 것이 목표입니다.

## Storyboard

1. 문제와 목표를 소개합니다.
   - 사용자가 식재료를 쉽게 등록하고 관리해야 하는 이유를 설명합니다.
   - FreshKitchen이 입력, 저장, 조회, AI 분석을 연결하는 방식을 설명합니다.
2. 전체 아키텍처를 짧게 설명합니다.
   - 프론트엔드가 사용자 입력과 확인 화면을 담당합니다.
   - 백엔드가 인증 사용자 기준 저장, catalog 매핑, storage 매핑을 담당합니다.
   - AI 서버가 이미지 classification과 OCR 분석을 담당합니다.
3. 핵심 사용자 흐름을 시연합니다.
   - 직접 식재료 등록
   - 홈에서 수동 식재료 입력
   - `D-1`, `D-3`, `D-5`, `D-10` 버튼을 사용한 유통기한 빠른 입력
   - Fresh 항목 중심의 유통기한 관리
   - 이미지 또는 영수증 기반 등록
   - 홈 요약, 목록, 상세 확인
4. 기술적으로 맞춘 지점을 설명합니다.
   - 프론트는 request DTO에서 내부 ID 노출을 줄이고, AI 채팅 UI에서는 카메라 기능을 제외합니다.
   - 백엔드는 Catalog를 seed master data 기준으로 운영하고, AI `category`를 fallback 데이터로 활용합니다.
   - AI는 이미지 classification, OCR, 냉장고 detection 결과를 제공합니다.
   - 실패 시 대체 흐름과 예시 응답을 준비했습니다.
5. 결과와 남은 개선점을 정리합니다.
   - 발표 기준으로 완성된 흐름을 정리합니다.
   - 후속 개선이 필요한 API, AI 정확도, 운영 배포 항목을 구분합니다.

## Demo Flow

1. 서비스 진입 화면을 보여줍니다.
2. 홈에서 수동 식재료 입력으로 진입합니다.
3. `D-1`, `D-3`, `D-5`, `D-10` 버튼을 사용해 유통기한을 빠르게 입력합니다.
4. 등록된 Fresh 항목이 목록 또는 상세 화면에 반영되는지 보여줍니다.
5. 이미지 또는 영수증 업로드를 시연합니다.
6. AI 인식 결과를 확인하고 저장 흐름으로 연결합니다.
7. 홈 요약에서 전체 상태를 확인합니다.
8. 실패 시 대체 자료가 준비되어 있음을 짧게 언급합니다.

## Checklist

### 프론트

- [ ] 발표용 계정과 시연 데이터가 준비되어 있습니다.
- [ ] 프론트엔드 실행 방법을 확인했습니다.
- [ ] 홈 manual 입력과 유통기한 빠른 입력 흐름을 설명할 수 있습니다.
- [ ] AI 채팅 UI에서 카메라 기능이 제외된 상태를 확인했습니다.

### 백엔드

- [ ] 백엔드 실행 방법을 확인했습니다.
- [ ] 문서의 API 계약과 실제 시연 흐름이 다르지 않습니다.

### AI

- [ ] AI 서버 실행 또는 대체 응답 방법을 확인했습니다.

### 공통/연결

- [ ] 실패 시 화면 캡처, API 응답 예시, 녹화 자료 중 하나를 사용할 수 있습니다.

## Fallback

실시간 시연이 실패하면 아래 순서로 대체합니다.

1. 미리 준비한 화면 캡처나 녹화 자료를 보여줍니다.
2. 백엔드 API 응답 예시를 직접 보여줍니다.
3. `api/examples/`의 요청과 응답 예시로 최종 흐름을 설명합니다.
