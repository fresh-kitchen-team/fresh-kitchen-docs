# 14주차 발표

Status: draft
Updated: 2026-05-17

## Purpose

14주차 발표는 프론트엔드, 백엔드, AI가 연결된 중간 시연 상태를 보여줍니다.

핵심은 식재료 등록, 조회, 스캔 흐름이 하나의 사용자 흐름으로 이어지는지 확인하는 것입니다.

## Storyboard

1. 사용자가 식재료를 등록해야 하는 상황을 설명합니다.
   - 직접 입력과 이미지/OCR 기반 입력을 모두 FreshKitchen의 입력 흐름으로 설명합니다.
2. 직접 등록 흐름을 보여줍니다.
   - 프론트는 `storageType`과 식재료명을 보냅니다.
   - 백엔드는 catalog master data와 사용자 storage로 매핑합니다.
3. 이미지 또는 영수증 스캔 흐름을 보여줍니다.
   - 프론트가 이미지를 업로드합니다.
   - 백엔드가 AI 서버를 호출합니다.
   - AI 결과를 백엔드가 프론트 확인 화면용 DTO로 정리합니다.
4. 조회 화면에서 결과를 확인합니다.
   - 목록과 상세 화면에서 `category`, `emoji`, `storageType`, 상태값이 일관되게 보이는지 확인합니다.

## Demo Flow

1. 홈 화면 또는 식재료 목록 화면을 보여줍니다.
2. 직접 식재료 등록을 진행합니다.
3. 등록된 식재료가 목록 또는 상세 화면에 반영되는지 보여줍니다.
4. 이미지 또는 영수증 업로드 흐름을 진행합니다.
5. AI 인식 결과를 확인 화면에서 보여줍니다.
6. 확인한 결과를 저장 흐름으로 연결합니다.

## Checklist

### 프론트

- [ ] 직접 등록 request가 목표 DTO 기준과 맞습니다.
- [ ] Scan API 응답이 프론트 확인 화면에 표시됩니다.
- [ ] 목록과 상세 화면에 `category`, `emoji`, `storageType`이 표시됩니다.

### 백엔드

- [ ] 실패 응답이 발생했을 때 사용자에게 설명 가능한 상태입니다.
- [ ] 시연용 사용자, storage, catalog, item 데이터가 준비되어 있습니다.

### AI

- [ ] 실시간 AI 호출 또는 대체 응답이 준비되어 있습니다.

### 공통/연결

- [ ] 확인한 스캔 결과를 저장 흐름으로 연결할 수 있습니다.

## Fallback

실시간 AI 호출이 실패하면 준비된 AI 응답 예시를 사용합니다.

화면 시연이 실패하면 API 응답 예시와 화면 캡처로 흐름을 설명합니다.
