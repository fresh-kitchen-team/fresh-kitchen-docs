## Purpose

이 문서는 FreshKitchen 발표 문서의 입구입니다.

발표는 12주차, 14주차, 16주차 최종 발표 총 3번만 관리합니다.

각 발표의 스토리보드, 시연 흐름, 체크리스트는 `PRESENTATION/` 폴더에서 따로 관리합니다.

- [14주차 발표](PRESENTATION/week-14.md)
- [16주차 최종 발표](PRESENTATION/week-16-final.md)

## Common Scope

발표에서 보여줄 핵심은 아래 기준으로 정합니다.

- 사용자가 FreshKitchen을 왜 사용하는지 설명합니다.
- 사용자가 식재료를 등록하고 관리하는 흐름을 보여줍니다.
- AI 기능이 프로젝트에서 어떤 가치를 만드는지 보여줍니다.
- 프론트엔드, 백엔드, AI 파트가 어떻게 연결되는지 보여줍니다.

## Common Demo Flow

각 발표의 세부 시연 흐름은 발표 단위 문서에서 관리합니다.

기본 흐름은 아래 순서를 기준으로 조정합니다.

1. 사용자가 서비스에 진입합니다.
2. 사용자가 식재료 정보를 입력하거나 이미지를 업로드합니다.
3. 백엔드가 데이터를 저장하고 필요한 경우 AI 서버에 요청합니다.
4. AI 서버 응답을 바탕으로 결과를 보여줍니다.
5. 사용자가 결과를 확인하고 다음 행동을 선택합니다.

## Fallback Plan

실시간 시연이 실패하면 아래 순서로 대체합니다.

1. 미리 준비한 요청과 응답 예시를 보여줍니다.
2. 백엔드 API 응답을 직접 보여줍니다.
3. 화면 캡처나 녹화 자료로 최종 결과를 설명합니다.

## Update Rule

발표 일정이나 발표 파일 구성이 바뀌면 이 파일을 갱신합니다.

각 발표의 스토리보드와 체크리스트는 `PRESENTATION/` 폴더의 해당 주차 파일을 갱신합니다.
