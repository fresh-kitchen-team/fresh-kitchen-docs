# Presentation

Status: draft
Updated: 2026-05-17

## Purpose

이 문서는 FreshKitchen 발표와 시연 준비 상태를 한 곳에서 확인하기 위한 문서입니다.

팀원은 이 파일을 보고 발표 범위, 발표 흐름, 시연 순서, 실패 시 대체 흐름을 확인합니다.

## Presentation Scope

발표에서 보여줄 핵심은 아래 기준으로 정합니다.

- 사용자가 FreshKitchen을 왜 사용하는지 설명합니다.
- 사용자가 식재료를 등록하고 관리하는 흐름을 보여줍니다.
- AI 기능이 프로젝트에서 어떤 가치를 만드는지 보여줍니다.
- 프론트엔드, 백엔드, AI 파트가 어떻게 연결되는지 보여줍니다.

## Demo Flow

시연 흐름은 팀 합의 후 구체화합니다.

1. 사용자가 서비스에 진입합니다.
2. 사용자가 식재료 정보를 입력하거나 이미지를 업로드합니다.
3. 백엔드가 데이터를 저장하고 필요한 경우 AI 서버에 요청합니다.
4. AI 서버 응답을 바탕으로 결과를 보여줍니다.
5. 사용자가 결과를 확인하고 다음 행동을 선택합니다.

## Speaker Notes

발표 담당자는 아직 확정되지 않았습니다.

담당자가 정해지면 아래 기준으로 내용을 채웁니다.

- 도입 설명
- 기능 설명
- 기술 구조 설명
- 시연 진행
- 마무리 설명

## Demo Checklist

- [ ] 발표용 계정을 준비했습니다.
- [ ] 시연용 데이터를 준비했습니다.
- [ ] 프론트엔드 실행 방법을 확인했습니다.
- [ ] 백엔드 실행 방법을 확인했습니다.
- [ ] AI 서버 실행 또는 대체 응답 방법을 확인했습니다.
- [ ] 실패 시 대체 화면이나 예시 응답을 준비했습니다.

## Fallback Plan

실시간 시연이 실패하면 아래 순서로 대체합니다.

1. 미리 준비한 요청과 응답 예시를 보여줍니다.
2. 백엔드 API 응답을 직접 보여줍니다.
3. 화면 캡처나 녹화 자료로 최종 결과를 설명합니다.

발표 내용이 바뀌면 이 파일을 직접 갱신합니다.

발표 스크립트, 시연 순서, 실패 대응 내용을 여러 파일로 나누지 않습니다.
