# Scope

Status: draft
Updated: 2026-05-17

## Purpose

이 문서는 FreshKitchen 프로젝트에서 현재 포함하는 범위와 제외하는 범위를 정리합니다.

팀원이 기능을 추가하거나 발표를 준비할 때, 이 문서를 기준으로 이번 단계에서 무엇을 보여줄지 판단합니다.

## Included

현재 포함 범위는 최종 발표 기준으로 구체화합니다.

- 사용자 인증 또는 사용자 식별 흐름
- 식재료 등록과 조회
- 식재료 이미지 또는 OCR 기반 입력 흐름
- AI 서버와 연결되는 추천 또는 분석 흐름
- 발표와 시연에 필요한 최소 API 흐름

## Final Presentation Scope

최종 발표까지 포함할 제품 범위는 아래 기준으로 관리합니다.

### 프론트

- Fresh 항목 관리 화면을 포함합니다.
  - 유제품처럼 유통기한이 예민한 식재료를 중점 관리 대상으로 보여줍니다.
  - 유통기한 기반으로 사용자가 빠르게 상태를 판단할 수 있어야 합니다.
- 식재료 등록 화면에 유통기한 빠른 입력을 포함합니다.
  - `D-1`, `D-3`, `D-5`, `D-10` 버튼으로 만료일을 빠르게 입력할 수 있게 합니다.
  - 버튼은 입력 편의 기능이며, 실제 저장 기준은 계산된 유통기한 날짜입니다.
- 홈 화면에 수동 식재료 입력 진입을 포함합니다.
  - 사용자는 홈에서 `MANUAL` 방식으로 식재료를 직접 등록할 수 있어야 합니다.
- scan navigation에 냉장고 detection 진입을 포함합니다.
  - 사용자는 스캔 흐름에서 냉장고 이미지 기반 detection으로 진입할 수 있어야 합니다.
- 유저 프로필 화면을 포함합니다.
  - 사용자는 본인 계정 또는 서비스 사용 정보를 확인할 수 있어야 합니다.
- 동의 약관 화면 또는 동의 흐름을 포함합니다.
  - 사용자는 서비스 사용에 필요한 약관 동의 상태를 확인하거나 진행할 수 있어야 합니다.

### 백엔드

- Catalog는 seed master data를 기본으로 운영합니다.
  - 운영 데이터에서 catalog 후보를 추립니다.
  - 후보는 검증 후 필요한 catalog 데이터로 추가합니다.
- AI 서버가 보내는 `category` 값은 catalog 매핑 fallback 데이터로 활용합니다.
  - `category`는 catalog 매핑이 애매하거나 실패할 때 참고하는 보조 데이터입니다.
  - 최종 catalog 확정은 백엔드 catalog 정책을 따릅니다.

### AI

- 냉장고 detection 결과를 프론트 확인 흐름에 연결할 수 있게 제공합니다.
- catalog fallback에 활용할 수 있는 `category` 값을 제공합니다.

### 공통/연결

- Scan 결과 확인 화면과 Item 생성 연결을 포함합니다.
  - 사용자는 스캔 결과를 확인한 뒤 식재료 등록 흐름으로 이어갈 수 있어야 합니다.
- AI 채팅 문의 보내기와 채팅방 삭제는 API 명세 작성 대상입니다.
  - 아직 확정 API 계약으로 문서화하지 않고, 최종 발표 스코프의 명세 필요 항목으로 둡니다.

## Excluded

현재 제외 범위는 팀 합의 후 구체화합니다.

- 운영 배포 자동화
- 결제, 광고, 외부 커머스 연동
- 복잡한 권한 관리
- 발표 시연에 필요하지 않은 관리자 기능
- 실제 상용 수준의 모니터링과 장애 대응 체계
- 프론트: AI 채팅 UI의 카메라 기능

## Unclear

아래 항목은 아직 확정되지 않았습니다.

- 발표에서 AI 기능을 어디까지 실시간으로 보여줄지
- 프론트엔드가 모든 API를 직접 시연할지, 일부는 백엔드 도구로 대체할지
- 시연 데이터는 실제 입력으로 만들지, 미리 준비한 데이터로 시작할지
- 공통/연결: AI 채팅 문의 보내기와 채팅방 삭제 API의 endpoint, request, response 형식

## Update Rule

범위가 바뀌면 이 파일을 직접 수정합니다.

새로운 논의용 문서를 만들기보다, 확정된 결과를 이 문서에 반영합니다.
